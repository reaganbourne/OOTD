Auth Flow — Token Lifecycle
1
Register / Login
POST credentials → hashed with bcrypt
→
2
Issue Tokens
access_token (JWT, 15 min) + refresh_token (JWT, 7 days)
→
3
Store Session
SHA-256 hash of refresh token saved to DB. Raw token → HttpOnly cookie
→
4
Auto-Refresh
On 401: POST /auth/refresh → new token pair, old session revoked
→
5
Protected Routes
get_current_user() decodes JWT, returns User or 401
